Security Policy
Supported Versions
The following versions of Taskflow are actively supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 5.1.x   | ✅ Supported       |
| 5.0.x   | ❌ Not supported   |
| 4.0.x   | ✅ Supported       |
| < 4.0   | ❌ Not supported   |

Reporting a Vulnerability
We take the security of Taskflow and its users seriously. If you believe you have found a vulnerability (e.g. in task creation, permissions, authentication, or data handling), please report it responsibly.

Where to report: Send an email to [security@taskflow.com] or open a private security advisory through our GitHub repository.

What to include:

A clear description of the vulnerability.

Steps to reproduce the issue.

Potential security risks or impacts.

(Optional) Suggestions for fixing the issue.

Response time:

We acknowledge reports within 48 hours.

You’ll receive progress updates at least every 7 days.

Resolution process:

Valid reports will be prioritized and fixed in the next patch release.

Security fixes may be backported to supported versions.

Recognition: If desired, we will credit you in release notes once the issue is resolved.
